## 理论推导

### 逻辑回归基本原理
逻辑回归是一种用于二分类问题的线性模型，通过Sigmoid函数将线性预测转换为概率值：

$$
P(y=1|x) = \sigma(z) = \frac{1}{1 + e^{-z}}
$$

其中 $z = w^Tx + b$，$w$ 是权重向量，$b$ 是偏置项。

### 损失函数：二元交叉熵
对于二分类问题，使用二元交叉熵损失函数：

$$
\mathcal{L}(y, \hat{y}) = -\frac{1}{m}\sum_{i=1}^m \left[y^{(i)}\log(\hat{y}^{(i)}) + (1-y^{(i)})\log(1-\hat{y}^{(i)})\right]
$$

其中 $m$ 是样本数量，$y^{(i)}$ 是真实标签，$\hat{y}^{(i)} = \sigma(z^{(i)})$ 是预测概率。

### 梯度计算
通过链式法则计算损失函数对参数的梯度：

1. **对输出的梯度**：
   $$
   \frac{\partial \mathcal{L}}{\partial z} = \hat{y} - y
   $$

2. **对权重 $w$ 的梯度**：
   $$
   \frac{\partial \mathcal{L}}{\partial w} = \frac{1}{m} (\hat{y} - y)^T X
   $$

3. **对偏置 $b$ 的梯度**：
   $$
   \frac{\partial \mathcal{L}}{\partial b} = \frac{1}{m} \sum_{i=1}^m (\hat{y}^{(i)} - y^{(i)})
   $$

### 参数更新
使用梯度下降法更新参数：

$$
w := w - \alpha \frac{\partial \mathcal{L}}{\partial w}
$$
$$
b := b - \alpha \frac{\partial \mathcal{L}}{\partial b}
$$

其中 $\alpha$ 是学习率。

## 代码实现

### 核心模型类：Logistic_Regression
模型实现位于 `src/model.py`，主要包含以下方法：

#### Sigmoid函数
```python
def sigmoid(x):
    # 使用 np.clip 防止数值溢出
    x = np.clip(x, -500, 500)
    return 1 / (1 + np.exp(-x))
```

#### 前向传播
```python
def forward(self, X):
    # X 形状: (batch_size, n_features)
    # self.w.T 形状: (n_features, 1)
    z = X @ self.w.T + self.b 
    y = sigmoid(z)
    return y
```

#### 训练方法
```python
def train(self, X, Y, lr):
    batch_size = X.shape[0]
    y = self.forward(X)
    
    # 1. 交叉熵损失
    epsilon = 1e-15
    CCE = -np.sum(Y * np.log(y + epsilon) + (1 - Y) * np.log(1 - y + epsilon)) / batch_size 
    
    # 2. 计算梯度
    dz = (y - Y) / batch_size  # 梯度简化为 y - Y
    dw = dz.T @ X              # 权重梯度
    db = np.sum(dz)            # 偏置梯度
    
    # 3. 更新参数
    self.w -= lr * dw
    self.b -= lr * db
    
    return CCE
```

### 3. Covtype森林覆盖数据集实验

#### 实验二：50,000样本规模（样本增加效果验证）
- **数据集**：sklearn的`fetch_covtype`
- **样本数**：50,000个样本（从581,012中采样，约8.6%）
- **特征数**：54个特征
- **类别**：类别1和类别2

**训练参数优化**：
- 批次大小：128（保持稳定）
- 训练轮次：150（增加到150轮）
- 学习率：0.01（保持）
- 测试集比例：20%

**训练结果**：
- 自己模型准确率：**77.66%**
- sklearn模型准确率：**77.64%**
- 混淆矩阵：
  ```
  [[3046 1190]
   [1044 4720]]
  ```
- 损失变化：184.28 → 147.27
- 总误分类数：2234个（占测试集的22.4%）

**样本增加效果分析**：
1. **准确率变化**：从78.15%略微下降到77.66%，但模型间差距缩小
2. **性能对比**：自实现模型从略差于sklearn变为略优于sklearn（77.66% vs 77.64%）
3. **训练稳定性**：更大的数据集使两种模型的性能更加接近，表明自实现模型在更大数据集上的泛化能力更强
4. **损失收敛**：初始损失更高（184.28），但相对下降幅度更大（37.01 vs 11.60）

### 性能对比表

### 分析

#### 1. 模型等效性验证
- 在相对简单的数据集（乳腺癌、Digits）上，自实现模型与sklearn模型性能相当
- 验证了梯度计算和参数更新的正确性

#### 2. 性能差异原因
- **乳腺癌数据集**：自实现模型略优（100% vs 99.12%），可能由于：
  - Mini-batch梯度下降的随机性带来更好的泛化
  - 学习率衰减策略的优化效果
- **Covtype数据集**：
  - **10k样本**：sklearn模型略优（78.30% vs 78.15%），可能由于：
    - sklearn使用了更复杂的优化算法（如L-BFGS）
    - 正则化项的缺失（自实现模型未添加L2正则化）
  - **50k样本**：自实现模型略优（77.66% vs 77.64%），表明：
    - 随着样本数量增加，Mini-batch梯度下降的优势更加明显
    - 更大的数据集减少了过拟合风险，使自实现模型的泛化能力提升
    - 样本多样性增加，减少了训练过程中的随机波动影响

#### 3. 训练效率
- **自实现模型**：手动控制训练过程，便于调试和理解
- **sklearn模型**：自动选择优化算法，收敛更快

### 核心差异点
1. **优化算法**：
   - 自实现：Mini-batch梯度下降
   - sklearn：默认使用L-BFGS或SAG等高级优化器

2. **正则化**：
   - 自实现：未实现L1/L2正则化
   - sklearn：默认包含L2正则化（可通过参数调整）

3. **收敛条件**：
   - 自实现：固定轮次训练
   - sklearn：基于收敛条件自动停止

## 结论

### 主要成果
1. **成功实现**：从零实现了逻辑回归模型，包括前向传播、损失计算、梯度下降等核心组件
2. **正确性验证**：通过与sklearn官方模型对比，验证了自实现模型的正确性
3. **泛化能力**：在不同规模和复杂度的数据集上均表现出良好的性能

### 技术亮点
1. **数值稳定性**：使用np.clip防止Sigmoid函数数值溢出
2. **梯度简化**：利用Sigmoid+CrossEntropy的数学性质，将梯度简化为y - Y
3. **灵活数据处理**：支持多种数据集加载和预处理
4. **完整可视化**：提供损失曲线、决策边界等多维度可视化

### 改进方向
1. **正则化实现**：添加L1/L2正则化项，提高模型泛化能力
2. **高级优化器**：实现动量法、Adam等高级优化算法
3. **多分类扩展**：扩展为Softmax回归，支持多分类问题
4. **早停机制**：实现基于验证集的早停机制，防止过拟合

### 实验意义
本实验不仅验证了逻辑回归的理论基础，还提供了完整的实现框架，可作为深度学习入门的重要实践。通过对比分析，深入理解了不同优化策略对模型性能的影响，为后续更复杂的模型实现奠定了基础。
