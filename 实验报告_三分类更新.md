## 理论推导

### 逻辑回归基本原理
逻辑回归是一种用于二分类问题的线性模型，通过Sigmoid函数将线性预测转换为概率值：

$$
P(y=1|x) = \sigma(z) = \frac{1}{1 + e^{-z}}
$$

其中 $z = w^Tx + b$，$w$ 是权重向量，$b$ 是偏置项。

### 多分类扩展：Softmax回归
对于多分类问题（三分类及以上），将Sigmoid函数扩展为Softmax函数：

$$
P(y=k|x) = \frac{e^{z_k}}{\sum_{j=1}^{C} e^{z_j}}
$$

其中 $z_k = w_k^Tx + b_k$ 是第 $k$ 类的线性预测，$C$ 是类别总数。

### 损失函数：交叉熵
- **二分类**：二元交叉熵
  $$
  \mathcal{L}(y, \hat{y}) = -\frac{1}{m}\sum_{i=1}^m \left[y^{(i)}\log(\hat{y}^{(i)}) + (1-y^{(i)})\log(1-\hat{y}^{(i)})\right]
  $$

- **多分类**：多类交叉熵
  $$
  \mathcal{L}(y, \hat{y}) = -\frac{1}{m}\sum_{i=1}^m \sum_{k=1}^{C} y_k^{(i)}\log(\hat{y}_k^{(i)})
  $$

其中 $m$ 是样本数量，$y^{(i)}$ 是真实标签，$\hat{y}^{(i)}$ 是预测概率。

### 梯度计算
通过链式法则计算损失函数对参数的梯度：

1. **二分类梯度**：
   $$
   \frac{\partial \mathcal{L}}{\partial z} = \hat{y} - y
   $$

2. **多分类梯度（Softmax+CrossEntropy）**：
   $$
   \frac{\partial \mathcal{L}}{\partial z_k} = \hat{y}_k - y_k
   $$

3. **对权重 $w$ 的梯度**：
   $$
   \frac{\partial \mathcal{L}}{\partial w} = \frac{1}{m} (\hat{y} - y)^T X
   $$

4. **对偏置 $b$ 的梯度**：
   $$
   \frac{\partial \mathcal{L}}{\partial b} = \frac{1}{m} \sum_{i=1}^m (\hat{y}^{(i)} - y^{(i)})
   $$

### 参数更新
使用梯度下降法更新参数：

$$
w := w - \alpha \frac{\partial \mathcal{L}}{\partial w}
$$
$$
b := b - \alpha \frac{\partial \mathcal{L}}{\partial b}
$$

其中 $\alpha$ 是学习率。

## 代码实现

### 核心模型类：Logistic_Regression（二分类）
模型实现位于 `src/model.py`，主要包含以下方法：

#### Sigmoid函数
```python
def sigmoid(x):
    # 使用 np.clip 防止数值溢出
    x = np.clip(x, -500, 500)
    return 1 / (1 + np.exp(-x))
```

#### 前向传播
```python
def forward(self, X):
    # X 形状: (batch_size, n_features)
    # self.w.T 形状: (n_features, 1)
    z = X @ self.w.T + self.b 
    y = sigmoid(z)
    return y
```

#### 训练方法
```python
def train(self, X, Y, lr):
    batch_size = X.shape[0]
    y = self.forward(X)
    
    # 1. 交叉熵损失
    epsilon = 1e-15
    CCE = -np.sum(Y * np.log(y + epsilon) + (1 - Y) * np.log(1 - y + epsilon)) / batch_size 
    
    # 2. 计算梯度
    dz = (y - Y) / batch_size  # 梯度简化为 y - Y
    dw = dz.T @ X              # 权重梯度
    db = np.sum(dz)            # 偏置梯度
    
    # 3. 更新参数
    self.w -= lr * dw
    self.b -= lr * db
    
    return CCE
```

### 多分类模型类：Multiclass_LogisticRegression
新增模型实现位于 `src/model_multiclass.py`，主要包含以下方法：

#### Softmax函数
```python
def softmax(x):
    # 数值稳定性优化：减去最大值
    x = np.clip(x, -500, 500)
    exp_x = np.exp(x - np.max(x, axis=-1, keepdims=True))
    return exp_x / np.sum(exp_x, axis=-1, keepdims=True)
```

#### 前向传播
```python
def forward(self, X):
    # z 形状: (batch_size, n_classes)
    z = X @ self.w.T + self.b.T
    y = softmax(z)
    return y
```

#### 训练方法
```python
def train(self, X, Y, lr):
    batch_size = X.shape[0]
    y_pred = self.forward(X)  # 形状: (batch_size, n_classes)
    
    # 计算交叉熵损失
    epsilon = 1e-15
    y_pred_clipped = np.clip(y_pred, epsilon, 1 - epsilon)
    CCE = -np.sum(Y * np.log(y_pred_clipped)) / batch_size
    
    # 计算梯度 (softmax + 交叉熵的梯度简化为 y_pred - Y)
    dz = (y_pred - Y) / batch_size
    
    # 更新参数
    dw = dz.T @ X
    db = np.sum(dz, axis=0, keepdims=True).T
    
    self.w -= lr * dw
    self.b -= lr * db
    
    return CCE
```

### 分析

#### 1. 模型等效性验证
- 在相对简单的数据集（乳腺癌、Digits）上，自实现模型与sklearn模型性能相当
- 在二分类Covtype数据集上，两种模型性能接近（差异<0.3%）
- **在三分类Covtype数据集上，自实现模型仍保持与sklearn模型接近的性能（差异0.27%）**，验证了多分类扩展的正确性

#### 2. 性能差异原因
- **乳腺癌数据集**：自实现模型略优（100% vs 99.12%），可能由于：
  - Mini-batch梯度下降的随机性带来更好的泛化
  - 学习率衰减策略的优化效果

- **Covtype数据集（二分类）**：
  - **10k样本**：sklearn模型略优（78.30% vs 78.15%），可能由于：
    - sklearn使用了更复杂的优化算法（如L-BFGS）
    - 正则化项的缺失（自实现模型未添加L2正则化）
  - **50k样本**：自实现模型略优（77.66% vs 77.64%），表明：
    - 随着样本数量增加，Mini-batch梯度下降的优势更加明显
    - 更大的数据集减少了过拟合风险，使自实现模型的泛化能力提升

- **Covtype数据集（三分类）**：
  - sklearn模型略优（77.66% vs 77.39%），可能由于：
    - 三分类任务复杂度更高，对优化算法更敏感
    - sklearn的优化器在多分类任务上可能更有效
    - 自实现模型未包含正则化，可能导致轻微过拟合

#### 3. 训练效率
- **自实现模型**：手动控制训练过程，便于调试和理解
- **sklearn模型**：自动选择优化算法，收敛更快

#### 4. 多分类任务性能
- **总体性能**：三分类任务准确率（77.39%-77.66%）与二分类（77.64%-78.30%）相比略有下降，符合预期
- **类别不平衡影响**：类别2（样本量6.6%）在两个模型中都获得了最高F1分数，表明样本量不是决定性能的唯一因素
- **模型一致性**：两个模型在三分类任务中表现出相似的类别性能模式，验证了模型的一致性

### 核心差异点
1. **优化算法**：
   - 自实现：Mini-batch梯度下降
   - sklearn：默认使用L-BFGS或SAG等高级优化器

2. **正则化**：
   - 自实现：未实现L1/L2正则化
   - sklearn：默认包含L2正则化（可通过参数调整）

3. **收敛条件**：
   - 自实现：固定轮次训练
   - sklearn：基于收敛条件自动停止

4. **多分类支持**：
   - 自实现：新实现多分类版本（`src/model_multiclass.py`）
   - sklearn：原生支持多分类

## 结论

### 主要成果
1. **成功实现**：从零实现了逻辑回归模型，包括前向传播、损失计算、梯度下降等核心组件
2. **正确性验证**：通过与sklearn官方模型对比，验证了自实现模型的正确性
3. **泛化能力**：在不同规模和复杂度的数据集上均表现出良好的性能
4. **多分类扩展**：**成功实现了从二分类到多分类的扩展**，在三分类任务上验证了softmax回归和多分类交叉熵损失函数的正确性

### 技术亮点
1. **数值稳定性**：使用np.clip防止Sigmoid/Softmax函数数值溢出
2. **梯度简化**：利用Sigmoid+CrossEntropy/Softmax+CrossEntropy的数学性质，将梯度简化为y_pred - y_true
3. **灵活数据处理**：支持多种数据集加载和预处理（二分类与三分类）
4. **完整可视化**：提供损失曲线、决策边界等多维度可视化
5. **扩展性**：模块化设计，便于从二分类扩展到多分类

### 改进方向
1. **正则化实现**：添加L1/L2正则化项，提高模型泛化能力
2. **高级优化器**：实现动量法、Adam等高级优化算法
3. **早停机制**：实现基于验证集的早停机制，防止过拟合
4. **多任务学习**：扩展为多标签分类或回归任务

### 实验意义
本实验不仅验证了逻辑回归的理论基础，还提供了完整的实现框架，可作为深度学习入门的重要实践。通过对比分析，深入理解了不同优化策略对模型性能的影响，特别是**成功完成了从二分类到多分类的扩展**，验证了softmax回归的理论正确性和实现可行性。实验结果为后续更复杂的模型实现（如神经网络）奠定了基础。
